# 系统分类与力量体系

> 本文档记录游戏中所有"系统"的设定框架与具体实例。
> 系统 = 开发者遗留的世界管理模块，是可绑定的权限实体。

---

## 一、概述

### 设计原则

1. **源于开发工具**：每一个系统都对应开发者曾经使用的某种"世界管理模块"。
2. **功能单一化趋势**：流传到世间的模块大多在流失过程中受损，功能趋于单一。
3. **品质决定上限**：同一类系统，品质越高，越接近原始开发工具的完整功能。
4. **无组合性**：宿主可以同时绑定多个系统，但各系统**独立运作**，彼此之间不存在联动、加成或协同效应。拥有多个系统等于拥有多个独立工具，而非一个有机整体。

## 二、分类体系

### 2.1 按功能模块分类（主分类）

| 大类 | 对应开发模块 | 核心功能 | 典型持有者身份 |
|------|-------------|---------|--------------|
| **战斗类** | 战斗模拟模块 | 经验获取、属性成长、技能解锁 | 冒险者、佣兵 |
| **情报类** | 调试与观测模块 | 信息探查、地图绘制、目标分析 | 斥候、学者、商人 |
| **生产类** | 内容编辑模块 | 物品制造、材料合成、附魔强化 | 工匠、炼金术师 |
| **空间类** | 场景管理模块 | 储物空间、短距传送、领域展开 | 旅行者、遗迹猎人 |
| **辅助类** | 系统管理模块 | 任务追踪、交易辅助、状态监控 | 各类持有者都可能拥有 |

### 2.2 按品质等级分类

品质反映系统的完整度与功能上限。

| 品质 | 名称 | 完整度 | 功能特征 | 稀有度 |
|------|------|--------|---------|--------|
| ★ | **残缺** | 10%-30% | 单一功能，有副作用或限制 | 较常见（遗迹中大量存在） |
| ★★ | **基础** | 40%-60% | 单一功能稳定运行 | 常见（主角的第一个系统） |
| ★★★ | **进阶** | 70%-85% | 核心功能 + 1-2个扩展模块 | 稀少 |
| ★★★★ | **完整** | 90%-95% | 多模块集成，接近原始工具 | 极稀少（全球仅数十个） |
| ★★★★★ | **原型** | 100% | 开发者原始版本，拥有管理员权限 | 传说（可能不足10个） |

### 2.3 按绑定特性分类

| 特性 | 说明 | 影响 |
|------|------|------|
| **可弃型** | 可主动丢弃，变为无主状态 | 可交易、可掠夺 |
| **锚定型** | 与宿主灵魂深度绑定，无法剥离 | 随宿主死亡而消亡 |
| **破损型** | 流失过程中受损，功能不稳定 | 可能获得意外能力，也可能失控 |
| **共鸣型** | 可与同类系统产生共振，临时提升能力 | 持有者集会时出现特殊现象 |

## 三、各分类详细设定

### 3.1 战斗类系统

> 对应开发者的"战斗模拟模块"，用于快速测试角色成长曲线与战斗平衡。

#### 设计方向

- 这类系统持有者拥有"杀怪获取经验→升级→全面提升属性"的能力。
- 普通人眼中的"天才"，实则是系统的计算优化在起作用。
- 系统会根据战斗数据实时调整宿主的属性分配建议。
- 系统之间无组合效应：即使同时持有战斗系统和情报系统，两者也不会产生协同加成。

#### 已知实例

##### 【基础·战斗升级系统】（主角初期持有）

| 属性 | 内容 |
|------|------|
| 品质 | ★★ 基础 |
| 完整度 | 约50% |
| 核心功能 | 击杀敌人获取经验值，积累后提升等级，全属性小幅增长 |
| 扩展功能 | 无 |
| 界面表现 | 半透明属性面板、经验条、等级数字 |
| 副作用/限制 | 升级所需经验固定增长，无优化路径；无法查看敌人详细信息 |
| 背景信息 | 最普及的战斗类系统，被认为是"标准款" |

##### 【进阶·成长加速系统】（红狼持有）

| 属性 | 内容 |
|------|------|
| 品质 | ★★★ 进阶 |
| 完整度 | 约75% |
| 核心功能 | 经验获取效率提升 + 升级时额外获得属性点 |
| 扩展功能 | 「战斗分析」——战后自动生成战斗数据报告，提示改进方向 |
| 界面表现 | 属性面板 + 经验条 + 战斗日志窗口 |
| 副作用/限制 | 经验获取效率提升有每日上限 |
| 背景信息 | 红狼的核心系统之一，解释了为何他能远超同辈 |

### 3.2 情报类系统

> 对应开发者的"调试与观测模块"，用于查看世界运行参数、对象属性、地图数据。

#### 设计方向

- 持有者是地下世界的"眼睛"，能看见他人看不见的信息。
- 普通人眼中的"先知"或"鉴定大师"。
- 高级情报系统可能窥见世界的"底层代码"。

#### 已知实例

> 暂无，待补充。

#### 构思参考

- **鉴定之眼系统**：查看物品、人物、怪物的属性面板
- **地图绘制系统**：自动记录经过的区域，标记资源点与危险区域
- **推演系统**：输入条件后，模拟事件发展的几种可能性
- **真视系统**：看破幻象、伪装、隐藏的系统持有者身份

### 3.3 生产类系统

> 对应开发者的"内容编辑模块"，用于创建、修改、强化世界中的物品与实体。

#### 设计方向

- 持有者是地下世界的"造物者"，能制造出超越时代工艺的装备与道具。
- 普通人眼中的"神匠"，其作品被贵族与强者追捧。
- 高级生产系统可能触及"创造规则"本身。

#### 已知实例

> 暂无，待补充。

#### 构思参考

- **锻造编辑系统**：可视化编辑装备属性，消耗材料实现"数值锻造"
- **炼金配方系统**：自动解析材料成分，推演最优配比
- **附魔注入系统**：将击杀怪物的某种特性提取为附魔属性
- **建筑构造系统**：快速搭建临时工事或改造地形（空间类交叉）

### 3.4 空间类系统

> 对应开发者的"场景管理模块"，用于管理坐标、传送、储物、领域等空间操作。

#### 设计方向

- 持有者是地下世界的"行者"，机动能力与生存能力极强。
- 普通人难以理解其"随身携带大量物资"或"瞬间移动"的能力。
- 高级空间系统可能创造独立次元或扭曲现实空间。

#### 已知实例

> 暂无，待补充。

#### 构思参考

- **次元储物空间**：独立于世界规则的存储空间
- **坐标传送系统**：记录去过地点，消耗资源实现传送
- **安全屋系统**：在任意地点展开一个临时庇护所
- **领域展开系统**：在局部区域改写空间规则（高阶，极稀少）

### 3.5 辅助类系统

> 对应开发者的"系统管理模块"，用于协调、监控、优化其他模块的运行。

#### 设计方向

- 不直接提供战斗力，但能极大提升持有者的运营效率与信息优势。
- 可以与战斗类系统同时持有，但两者之间不会产生任何协同效应。
- 某些辅助系统的持有者可能同时持有战斗系统，但两个系统各自独立运行。

- **系统诊断系统**：检测其他系统的类型与品质（持有者之间的"雷达"）

### 4.2 多系统共存规则（已确立）

> **规则**：一个宿主可以同时绑定多个系统，但各系统**完全独立运行**。
>
> - 无属性加成：战斗系统的攻击力不会因为持有情报系统而提升
> - 无联动机制：情报系统发现的弱点不会自动传递给战斗系统生成对应技能
> - 无界面融合：每个系统保持独立的界面面板
>
> 这意味着：
> - 拥有多个系统等于拥有更多工具，但每个工具的性能不会因此变强
> - 高级持有者的优势来自**每个系统本身的高品质**，而非"凑齐一套"的组合效果

## 五、术语表

| 术语 | 定义 |
|------|------|
| 系统持有者 | 绑定了至少一个系统的个体 |
| 无主系统 | 被丢弃后尚未被重新绑定的系统 |
| 系统残片 | 破损严重的系统，功能不全但可能含有用模块 |
| 界面 | 系统持有者视野中出现的只有自己能看见的操作面板 |
| 完整度 | 系统相对于原始开发模块的功能保留百分比 |
| 模块 | 系统中的某个独立功能单元，高品质系统可包含多个模块 |
